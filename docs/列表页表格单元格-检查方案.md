# 列表页表格单元格展示 — 检查方案

## 1. 目标与验收标准

### 1.1 目标

对 **shellder-web-console** 中所有以表格（`antd` `Table`）呈现数据的列表/子列表页面做统一 UX 检查，保证：

1. **单元格内容不换行**（单行展示，`white-space: nowrap`）。
2. **超出列宽时省略显示**（`text-overflow: ellipsis`）。
3. **悬停可见完整内容**（Tooltip 或等效能力；禁止仅依赖浏览器原生 `title` 作为最终方案，见 §3.2）。

### 1.2 验收标准（逐格判定）

| 编号 | 检查项 | 通过条件 |
|------|--------|----------|
| A | 不换行 | 任意数据行、任意文本列在 1280px / 1440px 视口下均无多行折行（操作列 Tag/Button 除外） |
| B | 省略 | 故意填入 ≥80 字符的字段时，列内出现 `…`，且行高与其它行一致 |
| C | 悬停全文 | 对已被省略的单元格，鼠标悬停 **≤300ms** 内出现完整文案 Tooltip |
| D | 未省略不打扰 | 内容未超出列宽时，悬停 **不** 弹出 Tooltip（避免噪音） |
| E | 操作列豁免 | 「操作」列、固定宽度的状态 `Tag`、图标按钮列可不省略，但仍需 `nowrap` |
| F | 表头 | 表头文案不换行；过长时可省略 + Tooltip（与数据列一致） |

### 1.3 不在本方案范围

- 工作台首页 `List` 卡片（`pages/console/page.tsx`）— 非表格列表。
- Copilot 对话页 `List`（`pages/copilot/page.tsx`）。
- ER 图画布节点（`ErDiagramFlowCanvas` 等）— 非 Table。
- 纯表单、纯详情页无表格区域。

## 2. 技术规范（Ant Design 5.22）

项目依赖：`antd ^5.22.5`。

### 2.1 推荐写法（纯 `dataIndex` 文本列）

```tsx
{
  title: '名称',
  dataIndex: 'name',
  width: 200,           // 建议显式宽度，便于 ellipsis 生效
  ellipsis: true,       // 内置省略 + 溢出时 Tooltip
  onCell: () => ({ style: { whiteSpace: 'nowrap' } }),
  onHeaderCell: () => ({ style: { whiteSpace: 'nowrap' } }),
}
```

表格级建议（列较多时）：

```tsx
<Table
  tableLayout="fixed"
  scroll={{ x: 'max-content' }}  // 或按业务设最小总宽
  ...
/>
```

### 2.2 自定义 `render` 列

自定义 `render` 会破坏列级 `ellipsis` 的自动 Tooltip，须二选一：

**方案 A — `Typography.Text`（与连接器页一致）**

```tsx
render: (v: string) => (
  <Typography.Text ellipsis={{ tooltip: v }}>
    {v}
  </Typography.Text>
)
```

**方案 B — 显式 `Tooltip`**

```tsx
import { Tooltip } from 'antd';

render: (v: string) => (
  <Tooltip title={v}>
    <span className="block max-w-full truncate">{v}</span>
  </Tooltip>
);
// 注意：未溢出时也显示 Tooltip 时，可用 Typography.Text ellipsis 的 tooltip 行为
```

链接列示例（可点击 + 省略）：

```tsx
render: (v: string, row) => (
  <Typography.Text ellipsis={{ tooltip: v }}>
    <a onClick={() => open(row)}>{v}</a>
  </Typography.Text>
)
```

### 2.3 反模式（检查时应标为不通过）

| 反模式 | 典型位置 | 问题 |
|--------|----------|------|
| `<Space wrap>` 展示多个 Tag | `tenants/page.tsx` 开通能力、`users/page.tsx` 角色/租户 | 整格换行、撑高行高 |
| 仅 `ellipsis: true` 无 `width` / `tableLayout="fixed"` | 多处列表 | 列被内容撑开，省略不触发 |
| 自定义 `render` 仍只设 `ellipsis: true` | 部分审计/工具页 | 有省略样式但悬停无全文 |
| 仅用 HTML `title={v}` | `connectors/page.tsx` 部分链接列 | 样式弱、延迟与可访问性不一致 |
| 长 JSON/多行 `pre` 直接放进单元格 | 能力演示、调试类表格 | 必然换行 |

### 2.4 建议后续统一封装（可选，非本次检查阻塞项）

新增 `components/console/tableEllipsis.tsx`：

- `nowrapCell()` → `onCell` / `onHeaderCell` 工厂（参考 `connectors/page.tsx`）。
- `ellipsisTextColumn(title, dataIndex, width?)` → 标准文本列配置。
- `EllipsisCell({ children, tooltip })` → 供自定义 `render` 复用。

## 3. 检查方法

### 3.1 环境

- 浏览器：Chrome / Edge 最新稳定版。
- 视口：**1280×800**、**1440×900** 各测一遍。
- 登录具备各菜单权限的租户管理员账号。

### 3.2 单页检查步骤

1. 打开路由，等待表格加载完成。
2. 若无长数据：用浏览器 DevTools 临时改 DOM，或将后端/ mock 数据中某字段改为 100+ 字符重复串。
3. 对 **每一列**（除操作列）执行：
   - 目视：是否单行、`…` 是否出现；
   - 悬停：是否出现完整 Tooltip（§1.2 C）；
   - 缩短列宽（拖动或 DevTools 改 `width`）：省略是否仍生效。
4. 切换分页、筛选、排序后重复 spot check（防止 `render` 分支遗漏）。
5. 在 1280 视口横向滚动表格（若有 `scroll.x`），确认固定列/操作列不被遮挡。

### 3.3 代码静态检查（辅助）

在 `shellder-web-console` 根目录执行：

```bash
# 所有 Table 所在文件
rg '<Table' src/pages src/components/console -g '*.tsx' -l

# 已声明 ellipsis 的文件（便于对照「未覆盖」页面）
rg 'ellipsis:\s*true' src -g '*.tsx' -l

# 可能导致换行的 wrap
rg 'Space.*wrap|flex-wrap' src/pages -g '*.tsx'
```

**判定**：文件含 `Table` 且存在文本/ID/路径/摘要类列，但既无 `ellipsis` 也无 `Typography.Text ellipsis` / `Tooltip` → 记入「待整改」。

### 3.4 回归检查清单（打印用）

每页填写：

```
路由：________  文件：________  检查人：________  日期：________

[ ] A 不换行  [ ] B 省略  [ ] C 悬停全文  [ ] D 未省略无 Tooltip  [ ] E 操作列  [ ] F 表头

问题列：________  备注：________
```

## 6. 特殊列处理约定

| 列类型 | 处理方式 |
|--------|----------|
| 枚举 / 状态 `Tag` | 固定 `width` + `nowrap`；文案短，可不 Tooltip |
| 多值 Tag（角色、能力） | 改为 `+N` 折叠 + Popover 展示全部，或仅展示首个 + Tooltip 列全量 |
| 日期时间 | `nowrap`；一般不需 ellipsis |
| 数字 / 优先级 | 右对齐、`nowrap` |
| 链接 / 按钮 | 文本部分 ellipsis；操作按钮不换行 |
| JSON / 代码片段 | 单行 `ellipsis` + Tooltip 全文；禁止 `<pre>` 多行 |
| 空值 `—` | 不显示 Tooltip |
