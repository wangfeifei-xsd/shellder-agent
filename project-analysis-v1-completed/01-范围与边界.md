## 1. V1 已完成范围

V1 第一版验收包含以下**接入面与管理面**（侧栏无「实验中」标注）：

| 类别 | 内容 |
|------|------|
| Web 管理后台 | 工作台、知识库、『查询型』配置、Prompt 管理、会话管理、业务调试、嵌入式 Copilot 管理、OpenAPI 管理、用户与权限、租户管理、系统设置 |
| 嵌入式 Copilot | 配置、嵌入预览、实际嵌入页 `/copilot` |
| OpenAPI | 对外 `/openapi/v1/*` + 管理端应用接入与文档 |
| 运行时链路 | Agent Runtime、四类业务能力 Handler（经会话调试 / Copilot / OpenAPI 触达） |
| 治理 | JWT + RBAC、租户隔离、平台 LLM 接入、Prompt SSOT（基础设施） |

### 1.1 已通过正式入口验证的业务能力

| 能力 | 正式配置入口 | 正式运行入口 |
|------|--------------|--------------|
| 问答型 | 知识库 + 系统设置/模型接入 | 知识库/问答测试、会话调试、Copilot、OpenAPI |
| 查询型 | 『查询型』配置（连接器→ER→工具→查询测试） | 同上 + 业务调试 |
| 操作型 | （实验中）工具管理 | 会话调试、Copilot、OpenAPI（需 Action Tool 已配置） |
| 流程型 | （实验中）工具/任务 | 同上（需 Workflow Tool 已配置） |

操作型、流程型的**管理配置**仍在实验中菜单；Runtime Handler 已实现，验收通过 Copilot / 会话调试 / OpenAPI 路径完成。

## 3. 相对初始方案的实现差异（全局）

| 主题 | 初始方案 | V1 现网 |
|------|----------|---------|
| 前端工程 | Next.js | **Vite + React + React Router**（`shellder-web-console`） |
| 知识库服务名 | `pathy-knowledge-server` | **wiki 知识库服务**（配置键 `knowledge.wikiBaseUrl`，见 `docs/knowledge-wiki-integration.md`） |
| 知识库菜单 | 「知识库管理与规则」合一 | **知识库**（正式）与 **规则**（实验中）拆分 |
| 查询型配置 | 合并在连接器/工具管理 | 独立顶级菜单 **『查询型』配置**，复用 connector/tool 页面与 API |
| SQL 表白名单 | 表白名单 + 字段白名单 | 现网为 **表/字段黑名单** + ER 数据范围列映射 |
| 业务调试 | 无独立菜单 | 新增 **能力演示**（与 Copilot 嵌入契约对齐） |
| Prompt 子菜单 | 列表/详情/对比/绑定等多页 | 合并为 **模板列表 + 详情页**（版本/试跑/绑定在详情内） |
| OpenAPI 调用日志 | 独立菜单 | 合并在 **应用详情页**「调用日志」卡片 |
| 租户详情/编辑/隔离 | 独立菜单项 | 路由存在（`/tenants/:id/*`），侧栏仅列列表与新建 |

## 5. 工程组成（不变）

| 工程 | 职责 |
|------|------|
| `shellder-web-console` | Web 管理后台 + Copilot 嵌入页 |
| `shellder-agent-server` | REST + SSE + 平台核心域 |
