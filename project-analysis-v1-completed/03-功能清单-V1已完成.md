### 1.2B 工具管理（V1 产品化，HTTP 类 Tool）

**菜单**：

| 子菜单 | 路由 | Tool 类型 |
|--------|------|-----------|
| 查询工具 | `/tools/http-query` | `http_query`（HTTP 业务只读查询） |
| 工具列表 | `/tools` | `action` / `workflow` / `notification` |

**流程型（workflow）编排**（工具列表页）：

- 新建 `workflow` Tool 后，通过 **流程编排** 弹窗配置多步流水线
- 子步骤可选：`query`（NL2SQL）、`http_query`、`action`、`notification`（**不支持** workflow 嵌套）
- 每步支持 **入参绑定**（`paramBindings`）：`fixed` 固定值、`previous_step` 引用前置步骤输出（`rows.0.xxx` 等）、`user_message` 留空由 LLM 补全
- 保存前经 **ER 列类型 / 参数类型交叉校验**（`workflow-step-validation.ts`）

**说明**：`ToolType.query`（NL2SQL）仍在 **『查询型』配置**；原 `/tools/sql` 已迁至 `/query/channel-debug`。

**要点**：

- 第五类 Tool **`http_query`**：独立菜单、独立列表 API、归属 action 能力
- **AI 润色**：`POST /api/v1/tools/http-query/polish-draft`
- **导出对接文档**：查询工具页生成 Markdown 集成说明
- **预制模板**：music_sing_v1 演示配置
- Runtime：**信号模式** + **路由绑定后 LLM 自然语言抽参**

**API 前缀**：`/api/v1/tools`（含 `GET /http-query`、`POST /http-query/polish-draft`、`POST /parse-signal`、`POST /:id/invoke`）

详见 [`modules/12-工具管理.md`](./modules/12-工具管理.md)。
