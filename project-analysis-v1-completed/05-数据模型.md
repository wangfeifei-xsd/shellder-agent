# 数据模型 — V1 已完成模块

> 完整定义见 `shellder-agent/shellder-agent-server/prisma/schema.prisma`；本节仅列**已验收功能直接读写**的表。

---

## 1. 租户与用户

| 表 | 用途 |
|----|------|
| `tenant` | 租户主数据；`config` JSON 含开通能力、隔离策略 |
| `user` | 平台账号 |
| `role` | 角色；`menus` / `modules` / `toolScopes` / `policy` JSON |
| `user_role` | 用户-角色 |
| `user_tenant` | 用户-租户（多租户） |

---

## 2. 会话与消息

| 表 | 用途 |
|----|------|
| `session` | 会话；`capabilityType`、`status`、租户/用户 |
| `message` | 消息；`role`、`content`、`metadata`（工具调用等） |

---

## 3. 知识库

| 表 | 用途 |
|----|------|
| `knowledge_base` | 租户知识库绑定；`wiki_prefix` |
| `system_config` | `knowledge.wikiBaseUrl`、`knowledge.wikiTimeoutMs` |

**说明**：`kb_data_source`、`kb_document`、`kb_chunk`、`kb_embedding_task` 等表在 schema 中保留，**自建向量路径已废弃**；内容存 wiki 外挂服务。

| 表 | 用途 |
|----|------|
| `kb_layer_processing_job` | 知识层异步编译/嵌入任务（worker 消费） |

---

## 4. 连接器与 Tool

| 表 | 用途 |
|----|------|
| `connector` | 连接器；`db_readonly`（查询型）、`http` / `notification`（连接器管理） |
| `connector_db_metadata` | 原始 schema、ER draft/published、数据范围 |
| `tool` | Tool 定义；`type=query`（NL2SQL）、`http_query`（HTTP 业务查询）、`action` / `notification` / `workflow` |

`tool.config` 按类型：`sql` / `httpQuery` / `http` / `workflow`（`steps[]` 含 `paramBindings`：fixed / previous_step / user_message）。

---

## 4A. 能力路由

| 表 | 用途 |
|----|------|
| `capability` | 四类能力目录；`dependent_tools` JSON |
| `routing_rule` | 能力内路由规则；`conditions` JSON（keywords/patterns/intents/toolKind）、`tool_ids` |

租户首次使用时由 `ensureDefaultCapabilities` 自动补齐基础 capability 记录。

---

## 5. Prompt

| 表 | 用途 |
|----|------|
| `prompt_template` | 逻辑键、role、category、scope |
| `prompt_version` | 版本正文；`draft` / `published` / `deprecated` |
| `prompt_binding` | 业务对象 → `prompt_key` 引用 |

---

## 6. Copilot 与 OpenAPI

| 表 | 用途 |
|----|------|
| `open_api_app` | 接入应用；clientId/secret、允许租户与能力 |
| `open_api_call_log` | 第三方调用日志 |
| `copilot_config` | Copilot 嵌入配置；关联 `open_api_app` |

---

## 7. 系统设置

| 表 | 用途 |
|----|------|
| `system_config` | KV 配置（基础、模型、LLM 密钥等） |
| `notification_template` | 通知模板 |

---

## 8. 工作台聚合（只读）

| 表 | 用途 |
|----|------|
| `tool_call_audit` | 工具调用审计 → 工作台 Tool 统计 |
| `approval` | 待确认 → 工作台卡片 + **审批中心**（已产品化） |
| `task` | 异常任务 → 工作台卡片；流程型执行状态 |

---

## 9. Redis / 队列

| 用途 | 说明 |
|------|------|
| BullMQ | 知识层文档处理、长任务、任务队列执行 |
| 会话/SSE | 流式推送状态 |

---

## 10. 规则与审批相关表（2026-06 产品化）

| 表 | 用途 |
|----|------|
| `rule` / `rule_hit` | Policy 显式规则与命中留痕（含 `[路由]` 快照） |
| `approval` | 待确认 / 审批记录；Copilot 与管理端共用 |

三类 `*_audit` 表（`tool_call_audit`、`user_action_audit`、`external_call_audit`）采集与 **审计中心** 四页查询已验收，见 [15-审计中心.md](./modules/15-审计中心.md)。

仍属实验中 UI 的表：`skill`、`skill_execution_log`。

> **说明**：`capability`、`routing_rule` 已纳入 V1 验收（能力路由菜单）。
