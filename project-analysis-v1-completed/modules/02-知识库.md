# 模块：知识库

| 属性 | 值 |
|------|-----|
| 初始方案 | 功能清单 §1.7（知识部分）/ 执行计划 11A |
| 菜单 key | `knowledge` |

---

## 子菜单与路由

| 子菜单 | 路由 | 功能摘要 |
|--------|------|----------|
| 知识库管理 | `/knowledge` | 租户知识库实例、wiki 服务连接、wiki_prefix |
| 知识层管理 | `/knowledge/layers` | raw/wiki/schema 层文件 CRUD、上传、embed |
| 存储结构 | `/knowledge/structure` | 四层目录树、空目录维护 |
| 媒体库 | `/knowledge/media` | 媒体上传/列表/引用/索引 |
| 问答测试 | `/knowledge/recall-test` | 召回 / 召回+LLM 双测试 |

**不包含**：规则配置、规则命中（已拆至实验中「规则」菜单）。

---

## 后端

| 组件 | 路径 | API |
|------|------|-----|
| 知识库 CRUD | `knowledge/knowledge.controller.ts` | `/api/v1/knowledge-bases` |
| wiki 代理 | `knowledge/knowledge-proxy.controller.ts` | `/api/v1/knowledge/*` |
| 连接配置 | `knowledge/knowledge-connection.service.ts` | `/api/v1/knowledge/connection` |
| 问答预览 | `business-capability/qa-preview.controller.ts` | `/api/v1/knowledge/dialogue/qa-preview` |

---

## 外部依赖

- **wiki 知识库服务**（初始文档称 pathy-knowledge-server）
- 配置：`system_config` → `knowledge.wikiBaseUrl`、`knowledge.wikiTimeoutMs`
- 租户隔离：代理层注入 `knowledge_base.wiki_prefix`

详见 `shellder-agent/docs/knowledge-wiki-integration.md`。

---

## 数据表

- `knowledge_base`（租户绑定）
- `kb_layer_processing_job`（异步任务）
- wiki 内容 **不在** Agent MySQL（外挂存储）

---

## 对照初始方案

| 项 | 差异 |
|----|------|
| 服务名称 | wiki 知识库服务；配置键非 PATHY 环境变量 |
| 菜单结构 | 规则拆出；知识库为独立顶级菜单 |
| 自建向量表 | schema 保留但 API 标记废弃，以 wiki 为准 |
| 问答测试 | 双卡片对齐 Runtime 两阶段链路 ✅ |
