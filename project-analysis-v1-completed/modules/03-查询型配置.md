# 模块：『查询型』配置

| 属性 | 值 |
|------|-----|
| 菜单 key | `query`（RBAC 映射 `connector` + 部分 API 需 `tool`） |

---

## 子菜单

| 子菜单 | 路由 | 实现方式 |
|--------|------|----------|
| 数据库连接器 | `/query/db-connectors` | 复用 `ConnectorListPage variant="db"` |
| 库表 ER 图 | `/query/db-er` | `connectors/db-schema/page.tsx`（React Flow） |
| 数据库连接工具 | `/query/db-channel-tools` | 复用 `ToolPage variant="queryOnly"` |
| 查询测试 | `/query/query-test` | 独立页，NL2SQL / SQL / E2E |
| 通道调试 | `/query/channel-debug` | 复用 `tools/sql/page.tsx`（原 `/tools/sql`） |

HTTP / 通知类连接器见 **[14-连接器管理.md](./14-连接器管理.md)**（`/connectors`）。

旧路由 `/connectors/db-schema` → redirect 至 `/query/db-er`。

---

## 后端模块

| 模块 | 职责 |
|------|------|
| `connector/` | 连接器 CRUD、连通性、introspect、ER CRUD/发布 |
| `tool/` | Query Tool CRUD、sql-test、nl2sql-preview、query-e2e-preview |
| `query/` | Nl2SqlService、QueryResultService、DataScopeResolve |

---

## 核心 API

```
GET/POST/PATCH/DELETE  /api/v1/connectors
POST                   /api/v1/connectors/:id/test
POST                   /api/v1/connectors/:id/introspect
GET/PUT                /api/v1/connectors/:id/er-diagram/*
POST                   /api/v1/connectors/:id/sql-test

GET/POST/PATCH/DELETE  /api/v1/tools
POST                   /api/v1/tools/:id/nl2sql-preview
POST                   /api/v1/tools/:id/query-e2e-preview
POST                   /api/v1/tools/:id/sql-test
```

---

## 配置顺序（接入必读）

1. 新建 **db_readonly** 连接器（一库一条）
2. 连通性测试 → 抽取表结构 → LLM 生成 ER → 人工校正 → **发布**
3. （可选）配置数据范围/限制字段
4. 新建 **query** 类型 Tool，绑定连接器
5. 查询测试页验证 NL2SQL 与 E2E

运行时能力详见 [`capabilities/查询型能力.md`](../capabilities/查询型能力.md)。

---

## 实现差异

- 早期将 SQL 工具放在「工具管理」；V1 **独立顶级菜单**，页面复用 connector/tool 组件
- ER、NL2SQL、LLM 解读、数据范围为详细方案后增补能力，**均已验收**
- 表白名单 → 现网 **黑名单** + ER 数据范围
