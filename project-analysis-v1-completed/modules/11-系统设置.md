# 模块：系统设置

| 属性 | 值 |
|------|-----|
| 初始方案 | 功能清单 §1.13 / 执行计划 18 |
| 菜单 key | `settings` |

---

## 子菜单

| 子菜单 | 路由 | config 分组 |
|--------|------|-------------|
| 基础配置 | `/settings` | 平台名称、Logo、默认超时/分页 |
| 模型接入 | `/settings/llm` | OpenAI 兼容 LLM（**独立 Controller**） |
| 模型与响应配置 | `/settings/model` | 流式、编排超时/重试 |
| 通知配置 | `/settings/notification` | 通知模板 |

---

## 模型接入（§1.13 重点）

| API | 说明 |
|-----|------|
| `GET/PUT /api/v1/settings/llm` | baseUrl、model、apiKey（加密）、timeout、maxTokens |
| `POST /api/v1/settings/llm/test` | 极小 Chat 连通性测试 |

- 存 `system_config` 表
- Agent Runtime、Copilot、问答测试 **统一**读取
- **不**代理 wiki/pathy 的 LLM 设置

---

## 其他 API

- `/api/v1/system-settings/configs` — 分组 KV
- `/api/v1/system-settings/notification-templates` — 通知模板 CRUD

---

## 后端

- `system-settings/system-settings.controller.ts`
- `llm/llm-settings.controller.ts`、`llm/llm.service.ts`

---

## 数据表

- `system_config`（含 `knowledge.wikiBaseUrl` 等）
- `notification_template`

---

## 对照初始方案

与 §1.13 一致；「模型接入」为实施规格 §4 明确要求，V1 已验收。能力级响应模板若指 LLM 话术，应通过 Prompt 管理 `prompt_key` 绑定，非本页长文本配置。
