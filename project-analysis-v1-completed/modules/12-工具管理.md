# 模块：工具管理

| 属性 | 值 |
|------|-----|
| 菜单 key | `tool`（RBAC `tool.manage`） |
| 状态 | **已产品化**（HTTP Invoker + `http_query` + AI 润色 + 对接文档导出） |

---

## 信息架构

工具管理负责 **HTTP 类 Tool** 的注册与调试；**数据库智能问数**（`ToolType.query`）仍在『查询型』配置维护。

```
工具管理
├── 查询工具        /tools/http-query     → ToolType.http_query（HTTP 业务只读查询）
├── 工具列表        /tools                → action / workflow / notification
└── （旧）/tools/sql → 重定向 /query/channel-debug

『查询型』配置
├── 数据库连接工具  /query/db-channel-tools  → ToolType.query（NL2SQL）
├── 查询测试        /query/query-test          → NL2SQL / SQL / E2E
└── 通道调试        /query/channel-debug       → NL2SQL 试跑（原 /tools/sql）
```

| Tool 类型 | 菜单 | 能力归属 | 执行层 |
|-----------|------|----------|--------|
| `http_query` | 查询工具 | **action** 能力 | `ToolInvocationService` → `HttpToolInvoker` |
| `action` / `notification` | 工具列表 | action 能力 | 同上 |
| `workflow` | 工具列表 | workflow 能力 | `WorkflowToolInvoker`（子步骤） |
| `query`（数据库） | 查询型配置 | query 能力 | `SqlToolService`（**不在本菜单**） |

**边界（D3）**：`ToolType.http_query` 归属 **action** 能力，**不是** `query` 能力。

## 配置模型摘要

### HTTP 业务查询（`config.httpQuery`）

| 字段 | 说明 |
|------|------|
| `toolCode` | 稳定业务编码，供 Prompt / 信号引用（`[a-z][a-z0-9_]*`） |
| `intentTags` | 意图标签数组（供路由 / NLU 扩展） |
| `priority` | 工具优先级（目录排序） |
| `parameters[]` | 入参定义 → 可推导 `inputSchema` |
| `invoke` | method、path、queryMapping、bodyMapping、timeoutMs |
| `response` | type（`text_reply` / `json_data` / **`play_audio`**）、successPath、fieldMapping、replyTextPath |

### 操作型（`config.http`）

- legacy：`method`、`path`、`headers`、`bodyTemplate`
- 增强：`queryMapping`、`bodyMapping`、`responseMapping`（含 `type`，与 Invoker 共用）

## 管理端能力（查询工具页 `/tools/http-query`）

| 能力 | 说明 |
|------|------|
| **新建 / 编辑** | 表单化配置 httpQuery 全字段；绑定 HTTP 连接器 |
| **AI 润色** | 调用 `polish-draft`，根据当前表单 + 可选补充说明，LLM 补全/润色配置 |
| **添入预制** | `HTTP_QUERY_PRESET`（music_sing_v1 演示，对齐 agent-plant 种子） |
| **导出对接文档** | 生成 Markdown（`{toolCode}-integration.md`），含 parameters、mapping、信号格式、Runtime 说明 |
| **invoke 调试** | 单步调用 Invoker |
| **信号解析调试** | 页内 parse-signal 试跑区 |

对接文档生成：`tools/http-query/http-query-dev-doc.ts`。

**说明**：AI 润色与对接文档导出**当前仅覆盖 `http_query`**；工具列表中的 action/notification 尚无对应能力。

## Runtime 触发（http_query）

1. **路由模式**：action 能力 Stage2 绑定 `http_query` toolIds；用户自然语言经 **LLM 抽参**（`resolveInvokeParams`）补全入参
2. **信号模式**：LLM 或用户输出 `[查询工具:tool_code {"param":"value"}]`，`HttpQueryTriggerService` 解析后 invoke

入参解析优先级：信号 JSON → 不足时 LLM 从自然语言抽取（路由绑定执行时通常无信号，走 LLM 补参）。

Prompt 目录 key：`action.http_query.catalog`（`prompt-keys.ts` + `project-sql/22-tool-http-query/`）。

## 接入顺序（HTTP 查询工具）

1. **连接器管理** → 新建 **http** 连接器并完成连通性测试（见 [14-连接器管理.md](./14-连接器管理.md)）
2. **查询工具** 页新建 `http_query` Tool（可「添入预制」或 **AI 润色** 加速配置）
3. 页内 **invoke** / **parse-signal** 调试；可选 **导出对接文档** 交付业务方
4. **能力路由** 为 action 绑定 toolIds / `toolKind=http_query`（可选）
5. **业务调试 → 能力演示** 选「操作型」端到端验证

## E2E 验收要点

- [ ] 新建 http 连接器 → 查询工具绑定 → invoke 成功
- [ ] AI 润色后保存 → 再次编辑字段一致
- [ ] 导出对接文档含 parameters / mapping / 信号示例
- [ ] 路由绑定 http_query 后，自然语言问句（无信号）可 LLM 抽参并执行
- [ ] 信号 `[查询工具:tool_code {...}]` 在能力演示 / Copilot 可触发
