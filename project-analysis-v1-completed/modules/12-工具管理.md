## 信息架构

工具管理负责 **HTTP 类 Tool** 的注册与调试；**数据库智能问数**（`ToolType.query`）仍在『查询型』配置维护。

```
工具管理
├── 查询工具        /tools/http-query     → ToolType.http_query（HTTP 业务只读查询）
├── 工具列表        /tools                → action / workflow / notification
└── （旧）/tools/sql → 重定向 /query/channel-debug

『查询型』配置
├── 数据库连接工具  /query/db-channel-tools  → ToolType.query（NL2SQL）
└── 通道调试        /query/channel-debug       → NL2SQL 试跑（原 /tools/sql）
```

| Tool 类型 | 菜单 | 能力归属 | 执行层 |
|-----------|------|----------|--------|
| `http_query` | 查询工具 | **action** 能力 | `ToolInvocationService` → `HttpToolInvoker` |
| `action` / `notification` | 工具列表 | action 能力 | 同上 |
| `workflow` | 工具列表 | workflow 能力 | `WorkflowToolInvoker`（子步骤） |
| `query`（数据库） | 查询型配置 | query 能力 | `SqlToolService`（**不在本菜单**） |

## 核心 API

```
GET/POST/PATCH/DELETE  /api/v1/tools
GET                    /api/v1/tools/http-query          # http_query 列表
POST                   /api/v1/tools/parse-signal        # 解析 [查询工具:tool_code {...}]
POST                   /api/v1/tools/:id/test            # 通用测试
POST                   /api/v1/tools/:id/invoke          # 直连 Invoker 调试
```

数据库 query 专用（不归 HTTP 查询工具）：

```
POST  /api/v1/tools/:id/nl2sql-preview
POST  /api/v1/tools/:id/query-e2e-preview
POST  /api/v1/tools/:id/sql-test
```

## 配置模型摘要

### HTTP 业务查询（`config.httpQuery`）

- `toolCode`：Prompt / 信号引用
- `parameters[]` → 可推导 `inputSchema`
- `invoke`：method、path、queryMapping、bodyMapping
- `response`：successPath、fieldMapping、replyTextPath

### 操作型（`config.http`）

- legacy：`method`、`path`、`headers`、`bodyTemplate`
- 增强：`queryMapping`、`bodyMapping`、`responseMapping`（与 Invoker 共用）

---

## Runtime 触发（http_query）

1. **路由模式**：action 能力 Stage2 绑定 `http_query` toolIds
2. **信号模式**：LLM 输出 `[查询工具:tool_code {"param":"value"}]`，`HttpQueryTriggerService` 解析后 invoke

Prompt 目录 key：`action.http_query.catalog`（`prompt-keys.ts` + seed）。

---

## 接入顺序（HTTP 查询工具）

1. 新建 **http** 连接器（工具管理仍依赖连接器管理中的 HTTP 连接器，菜单可仍为实验中）
2. **查询工具** 页新建 `http_query` Tool，配置 invoke / response
3. 页内 **invoke** 或 **parse-signal** 调试
4. **业务调试 → 能力演示** 选「操作型」定向验证（或能力路由绑定 toolIds）

---

## 对照初始方案

- 初始方案将 SQL 工具与 HTTP 工具混在同一「工具管理」；V1 将 **数据库 query** 拆至『查询型』配置
- 改造后新增 **HTTP 业务查询**（`http_query`）与 **统一 Invoker**，与 agent-plant QueryTool 语义对齐
- `/tools/sql` 已迁至 `/query/channel-debug`
