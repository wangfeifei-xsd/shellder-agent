## 信息架构

```
能力路由
├── 路由规则        /routing/rules           → 能力内 routing_rule（keywords/patterns/toolIds/toolKind）
├── 路由测试        /routing/test            → Stage1 + Stage2 联调（含模拟定向）
└── 能力目录        /routing/capabilities    → 四类能力 + dependentTools
```

| 能力 type | Tool 层 | 说明 |
|-----------|---------|------|
| `qa` | 无 Tool | 问答型，不走 Tool 编排 |
| `query` | `ToolType.query` | **仅 NL2SQL**（智能问数） |
| `action` | `action` / `notification` / **`http_query`** | HTTP 写操作 + HTTP 业务只读查询 |
| `workflow` | `workflow` | 流程型 |

**边界（D3）**：`ToolType.http_query` 归属 **action** 能力，**不是** `query` 能力。

---

## 两阶段路由

| 阶段 | 方法 | 职责 |
|------|------|------|
| Stage1 | `routeCapability()` | 跨类型 rules + 启发式；可选 LLM（`routing.classify.system`，默认关） |
| Stage2 | `routeWithinCapability()` | 能力内 `routing_rule` → toolIds；action 内可选 HTTP 查询信号 |
| 编排 | `routeFull()` | Runtime / 路由测试统一入口 |

**Copilot routingMode**（`copilot_config.features`）：

| mode | 创建会话 | 首条消息 |
|------|----------|----------|
| `auto`（默认） | capabilityType 可不传 | Stage1 自动 + Stage2 |
| `pinned` | capabilityType **必填** | 跳过 Stage1 + Stage2 |
| `hybrid` | 可不传 | Tab 可覆盖 Stage1 |

## 核心 API

```
GET/POST/PATCH/DELETE  /api/v1/routing-rules
GET/POST/PATCH/DELETE  /api/v1/capabilities
POST                   /api/v1/routing/test              # 全链路（typeStage + intraStage）
POST                   /api/v1/routing/test/intra        # 仅 Stage2
POST                   /api/v1/routing-rules/ai-suggest   # AI 辅助生成规则草案
```

Runtime 内调用 `RoutingEngineService.routeFull()`；结果写入 `routing_result` 系统消息（含 `typeStage` / `intraStage`）。

---

## `resolveToolIds` 优先级

1. `intraStage.toolIds`（能力内规则或信号命中）
2. `candidates[].toolIds`（Stage1 附带，兼容）
3. `capability.dependent_tools`
4. 按 `capabilityType` + `toolKind` fallback 第一个 enabled Tool

action 分支：`toolKind` 为 `http_query` / `action` / `notification` 时分流。

---

## 路由规则 conditions 扩展

```json
{
  "keywords": ["订单"],
  "patterns": ["^查.*订单"],
  "toolKind": "http_query"
}
```

`toolKind` 仅 **action** 能力规则可选；用于 Stage2 绑定具体 Tool 类型。

## LLM Stage1（可选）

- Prompt key：`routing.classify.system`（`prompt-keys.ts` + `project-sql/23-routing-llm-classify/`）
- 开启：`tenant.config.routing.llmClassifyEnabled = true` 或 `ROUTING_LLM_CLASSIFY_ENABLED=true`
- 关闭时行为等同规则 + 启发式（Phase 1）

## E2E 验收清单

### 路由测试页

- [ ] 无 `pinnedCapabilityType`：输入 NL2SQL 问句 → Stage1=`query`，Stage2 命中 query Tool 规则（若有）
- [ ] 开启「模拟定向」+ `action`：HTTP 业务问句 → Stage2 命中 `http_query` toolIds（规则 `toolKind` 或信号）
- [ ] `/routing/test` 与 Runtime 对同一输入的 `typeStage` + `intraStage` 一致

### Copilot 嵌入

- [ ] **auto**（默认 `routingMode`）：无需先点 Tab，首条消息自动路由并展示识别 Banner
- [ ] **pinned**：创建会话须传 `capabilityType`；Stage2 能力内规则可命中 toolIds
- [ ] 低置信（可选）：`clarifyOnLowConfidence=true` 时弹出确认 Modal

### 与工具方案联调

- [ ] 「查订单列表」类 NL2SQL → **query** 能力，无回归
- [ ] HTTP 业务查询（规则或 `[查询工具:tool_code {...}]` 信号）→ **action** + `http_query` Tool 执行
- [ ] Policy `needConfirmation` 在路由级与 Tool 级不退化

### OpenAPI 对齐

- [ ] 创建会话 `capabilityType` optional；未传则首条消息 `routeFull` 自动判定
