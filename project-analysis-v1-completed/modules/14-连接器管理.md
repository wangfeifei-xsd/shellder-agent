# 模块：连接器管理

| 属性 | 值 |
|------|-----|
| 菜单 key | `connector`（RBAC `connector.manage`） |
| 状态 | **已产品化**（HTTP / 通知类连接器独立菜单；只读库仍归『查询型』配置） |

---

## 信息架构

连接器按 **类型分两个管理入口**，共用 `ConnectorListPage` 与 `/api/v1/connectors` API：

```
连接器管理（正式菜单）
└── 连接器列表    /connectors          → type ∈ { http, notification }

『查询型』配置
└── 数据库连接器  /query/db-connectors  → type = db_readonly（variant="db"）
```

| 连接器 type | 管理入口 | 用途 |
|-------------|----------|------|
| `http` | `/connectors` | action / http_query / workflow 外部 HTTP |
| `notification` | `/connectors` | notification Tool、流程型通知步骤 |
| `db_readonly` | `/query/db-connectors` | NL2SQL / SQL Query Tool（**不经 HTTP 查数**） |

**边界**：`/connectors` 页明确提示「只读数据库请在『查询型』配置维护」，避免与 HTTP 连接器混配。

旧路由 `/connectors/db-schema` → redirect 至 `/query/db-er`。

## 后端模块

| 路径 | 职责 |
|------|------|
| `connector/connector.service.ts` | CRUD、凭证加密、连通性测试、allowedToolScopes 校验 |
| `connector/connector.controller.ts` | REST API |
| `connector/connector-schema.service.ts` | introspect、ER CRUD/发布（查询型） |
| `connector/connector-schema.controller.ts` | `/api/v1/connectors/:id/*` ER 相关 |
| `connector/connector-sql-test.service.ts` | 只读库 SQL 测试 |
| `connector/connector-introspection.service.ts` | information_schema 抽取 |

---

## 核心 API

```
GET/POST/PATCH/DELETE  /api/v1/connectors
GET                    /api/v1/connectors/db-schema     # 租户库表结构汇总（查询型 ER 入口）
POST                   /api/v1/connectors/:id/test        # 连通性测试
POST                   /api/v1/connectors/:id/introspect  # db_readonly 表结构抽取
POST                   /api/v1/connectors/:id/sql-test    # 只读库 SQL 试跑
GET/PUT                /api/v1/connectors/:id/er-diagram/*
```

权限：`@RequireAnyMenu('query', 'connector')`（查询型与连接器管理菜单均可访问 API）。

## 前端实现

| 路由 | 实现 |
|------|------|
| `/connectors` | `connectors/page.tsx` → `ConnectorListPage variant="general"` |
| `/query/db-connectors` | `query/db-connectors/page.tsx` → `variant="db"` |

`variant` 差异：类型选项、列表过滤、页面文案（section / alert / empty）。

---

## 接入顺序（操作型 / HTTP 查询）

1. **连接器管理** → 新建 `http` 连接器（target URL、auth、allowedToolScopes）
2. 连通性测试通过
3. **工具管理 → 查询工具** 或 **工具列表** 新建 Tool 并绑定该连接器
4. 见 [`12-工具管理.md`](./12-工具管理.md)、[`capabilities/操作型能力.md`](../capabilities/操作型能力.md)

查询型（NL2SQL）接入顺序见 [`03-查询型配置.md`](./03-查询型配置.md)。

---

## 实现差异

|----------|------|
| 连接器与工具管理合一 | **双入口**：HTTP/通知独立菜单；只读库在『查询型』配置 |
| 独立连接器管理菜单 | 曾标注实验中；**现已产品化**（侧栏无「实验中」） |
| 一连接器一库（db_readonly） | 不变，`target` + `config.properties.database` |
