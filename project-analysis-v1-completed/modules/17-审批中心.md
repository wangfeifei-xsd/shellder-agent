---

## 与 Runtime / Copilot 联动

1. Tool `needConfirmation`、Policy 规则 `need_confirm`、或 **路由规则 `needConfirmation`** → 创建 `approval`，会话/任务 `pending_confirm`
2. **确认执行** → `ApprovalRuntimeService` → `AgentRuntimeService.resumeFromApproval` 断点续跑
3. **驳回** → 任务/会话失败，写入驳回原因
4. 超时 → `job-worker` 标记 `approval.timeout`

### Copilot 嵌入（`/copilot`）

| 能力 | API |
|------|-----|
| 待确认列表 | `GET /copilot/v1/confirmations` |
| 确认 / 驳回 | `POST /copilot/v1/confirmations/:id` |

**现网修复（2026-06）**：

- 对话内 **确认执行** 按钮：SSE 未带 `approvalId` 时按 session 从待确认列表补查
- Copilot 审批人 ID 非 UUID 时仅写 `reviewer_name`，避免 `reviewer_id` 写库失败
- 内联确认区采用聊天气泡式卡片（非 Alert 条）

管理端审批详情路由 `/approvals/:id` 与 Copilot 内联确认共用 `ApprovalRuntimeService`。
