# 17 — 会话管理与调试台（Phase 16）

## 模块说明

本阶段为 Web 管理后台深化，补全 会话管理模块的完整 Web 功能：

- **会话详情页**：完整消息流（工具调用节点、确认节点）、会话上下文摘要、关联任务列表
- **消息记录页**：按消息类型筛选、时间线视图
- **调试台**：创建调试会话、发送消息、SSE 实时事件轨迹、路由结果展示、Tool 选择/入参/出参/耗时

## 依赖的前序模块

| 序号 | 模块 | 依赖内容 |
|------|------|----------|
| 08 | 会话与消息核心 | `session` 表、`message` 表 |
| 09 | 任务中心与异步 Worker | `task`、`task_step`、`task_log` 表 |
| 10 | 能力路由 | `capability`、`routing_rule` 表 |
| 11 | 技能书管理 | 技能书命中结果展示 |
| 11A | 知识库代理 | 问答召回测试 |
| 12 | Agent 运行时 | SSE 流式响应、发送消息 API |
| 13 | 四类业务能力 | 能力执行结果展示 |

## SQL 变更

**本阶段无数据库结构变更。**

后端 API 主要在 08–13 已具备；本阶段聚焦 UI/UX 与联调。所有功能基于已有的 `session`、`message`、`task` 等表实现。

## 新增后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/sessions/:id/messages` | 消息记录列表（支持按类型筛选、分页） |
| POST | `/api/v1/sessions/debug` | 创建调试会话（指定租户、场景） |

## 增强后端 API

| 方法 | 路径 | 变更说明 |
|------|------|----------|
| GET | `/api/v1/sessions/:id` | 返回结果新增 `tasks` 字段（关联任务列表） |

## 新增前端页面

| 路径 | 说明 |
|------|------|
| `/sessions/[id]` | 会话详情页（完整消息时间线、关联任务） |
| `/sessions/messages` | 消息记录页（按类型筛选、时间线视图） |
| `/sessions/debug` | 调试台（创建调试会话、发送消息、SSE 轨迹） |

## 执行顺序

1. 本阶段无 SQL 需执行
2. 后端代码变更后重启 `shellder-agent-server`
3. 前端代码变更后重启 `shellder-web-console`

## 注意事项

- Prisma schema 无变更，无需执行 migration
- 调试会话本质上是普通 Session，标题以 `[调试]` 为前缀区分
- SSE 事件轨迹仅在调试台前端展示，不额外持久化
